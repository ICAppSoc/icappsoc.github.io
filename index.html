iOS Session One

Delegate pattern + UITableView

You can download the slides over here. We hope that the session was useful and you got a basic understanding of the delegate pattern and its use inside iOS applications. Here you can find another brief coverage of the pattern by Apple and for the UITableView be sure to check out the Delegate and the Data Source.

Regarding the Weather app, these are the steps needed to complete the project:

  • Create a new XCode project using the Single View Application template
  • Open Main.storyboard and drag a UITableView inside the view
  • By control-dragging the table view to ViewController.h you should be able to create a new property for the view controller, call it tableView
  • Control drag from the table view again to the ViewController cell which is top left in your storyboard window, and make the view controller both the Delegate and the Data Source for the table view. (You must do this action twice)
  • In ViewController.h declare that the class is going to conform to UITableViewDelegate and UITableViewDataSource (this can also be done in ViewController.m, in the private interface)
  • Create a new property for the ViewController which is going to be a mutable array to hold the various forecasts. Call it daysWeather.
  • Write down the required methods for the UITableViewDataSource (number of rows and the cell for each index path).
  • Go to this page and grab the method in the gist. Paste it in ViewController.m and in the viewDidLoad method call it. ([self downloadWeatherForecast];)
  • Specify the exact content of the text label of the UITableViewCell(s) in your table view based on the content of the array. For each element you can access the name of the day using dayWeather[@"name"] and the forecast with dayWeather[@"forecast"][@"weather"][0][@"main"], where dayWeather is one dictionaries in the daysWeather array.
  • Add the did select cell method and prompt an UIAlertView, then have fun adding more cool features!
The entire code can be found on GitHub at this address.

Android Session One - The Force Awakens

Please download and install the following tools, in order:

Java Development Kit 7+ - once installed, select Start menu > Computer > System Properties > Advanced System Properties. Then open Advanced tab > Environment Variables and add a new system variable JAVA_HOME that points to your JDK folder, for example C:\Program Files\Java\jdk1.7.0_40.

Android Studio - version 1.0 is finally out and comes with a user-friendly installer.

To start developing, launch the Android SDK Manager and install:

Under Tools

  • Android SDK Tools
  • Android SDK Platform-tools
  • Android SDK Build-tools

Under Android 5.0

  • SDK Platform
  • Intel x86 Atom System Image
  • Google APIs

Under Extras

  • Android Support Repository
  • Android Support Library
  • Intel x86 Emulator Accelerator (HAXM)

To run applications, you have two choices:

Using physical Android device

Using virtual Android device (AVD)

  1. Go to %ANDROID_SDK%/extras/intel/HAXM and run the installer.
  2. Find and run the Android Virtual Device (AVD) Manager.
  3. Go through the wizard to create a virtual device, choosing the x86 image.

Then just click Run in Android Studio, and if the project is error-free, it will let you choose your hardware device or launch your emulator.
